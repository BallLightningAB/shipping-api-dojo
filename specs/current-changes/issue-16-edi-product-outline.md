# Issue 16 EDI Product Outline

Date: 2026-04-12
Issue: [#16](https://github.com/BallLightningAB/shipping-api-dojo/issues/16)
Branch: `codex/issue-16-edi-product`
Scope: Product strategy follow-on. No EDI implementation in issue `#9`.

## Goal

Decide whether EDI should launch as a sibling product under API Dojo rather than as a new track inside Shipping API Dojo.

## Current Recommendation

- treat EDI as a sibling product such as `edi.apidojo.app`
- keep Shipping API Dojo focused on API-based carrier integrations
- share brand/system patterns where useful
- keep product-specific SEO, taxonomy, and curriculum architecture separate

## Outline

1. Define the EDI product boundary and target audience.
2. Define the initial curriculum scope around X12, EDIFACT, acknowledgements, transport, mapping, and operations.
3. Decide the information architecture, naming, and domain strategy.
4. Define which shared systems can be reused across products.
5. Define cross-linking rules between Shipping API Dojo and the future EDI product.
